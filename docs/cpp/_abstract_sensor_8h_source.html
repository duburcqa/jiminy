34  ///
35  /// \brief Structure holding the data for every sensors of a given type.
36  ///
37  /// \details Every sensors of a given type must have the same 'behavior', e.g. the same
38  /// delay interpolation order and output type. However, their physical properties
39  /// may defer, such as the delay, the noise level or the bias. This enable us to
40  /// optimize the efficiency of data storage by gathering the state of every sensor
41  /// of the given type in Eigen Vectors by simply adding an extra dimension
42  /// corresponding to the sensor ID.
43  ///
44  ///////////////////////////////////////////////////////////////////////////////////////////////
