3 Jiminy is an open-source C++ simulator of poly-articulated systems. It is maintained by [Wandercraft Team](https://www.wandercraft.eu/en/), and available on [Github](https://github.com/Wandercraft/jiminy).

5 This page contains only the Doxygen documentation for Python code. It is automatically generated. A README is available on the Github. Take a look to the [Wiki](https://github.com/Wandercraft/jiminy/wiki) for more information.

7 ## Usage
8 
9  This documentation groups together all the native Python code and bindings, that is part of the jiminy project.
10  The source code is split in several python packages, each containing both source code for the package
11  and command line scripts, both of which are covered here.
12 
13  - For a list of command-line tools (standalone scripts that can be called from the command line), refer to the
14  [Command Line Scripts](./group__scripts.html) page.
15  - Documentation for the package sources is available by three different tabs:
16  - [Packages](./namespaces.html), where file are sorted by python package.
17  - [Classes](./annotated.html), where only python classes are listed.
18  - [Files](./files.html), where one may browse the file directory tree directly.
19  This is the most complete reference (i.e. some file might be inaccessible from the other two interfaces).
20  - Python bindings of the C++ source code are documented as well: typically look for <ClassName>Visitor class.

22 ## Contributing to this documentation
23 
24  Here are some guidelines on how this documentation is generated:
25 
26  - doxygen is used to parse the documentation, thus everything works like in C code:
27  - use triple-double quote comment (""") to start and end a doxygen comment block.
28  - comments must be placed after the declaration of the function they comment, or the class creation.
29  - all [special commands](https://www.stack.nl/~dimitri/doxygen/manual/commands.html) are available. For declaring
30  a command, \\ or \@ character can be used: for backward compatibility with previously written code, please use
31  \@ symbol only in python. Here is an example:
32 
33  def answer(the_question):
34  """
35  \@brief Gives the answer to The question.
36  \@details This function gives the answer life, the universe and everything.
37  \@note Not everybody might be psychologically ready for the answer.
38  \@param question The question.
39  \@return 42
40  """
41  return 42
42 
43  - File parsed: the following file are parsed:
44  - all python files (files with extension .py) in jiminy, **except** \__init__.py and setup.py files and
45  files in a build (sub)directory or in a unit (sub)directory.
46  - all files in a scripts directory, regardless of extension (command line scripts have no extensions). Files
47  without extensions are considered to be python scripts.
48  - .h files inside a directory named python.
49 
50  - To add a file to the [Command Line Scripts](./group__scripts.html) page, simply add `\@ingroup scripts` at the top of
51  the file.
