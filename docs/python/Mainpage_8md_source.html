5 ## Usage
6  This documentation groups together all the native Python code and bindings, that is part of the jiminy project.
7  The source code is split in several python packages, each containing both source code for the package
8  and command line scripts, both of which are covered here.
9 
10  - For a list of command-line tools (standalone scripts that can be called from the command line), refer to the
11  [Command Line Scripts](./group__scripts.html) page.
12  - Documentation for the package sources is available by three different tabs:
13  - [Packages](./namespaces.html), where file are sorted by python package.
14  - [Classes](./annotated.html), where only python classes are listed.
15  - [Files](./files.html), where one may browse the file directory tree directly.
16  This is the most complete reference (i.e. some file might be inaccessible from the other two interfaces).
17  - Python bindings of the C++ source code are documented as well: typically look for <ClassName>Visitor class.

19 ## Contributing to this documentation
20 
21  Here are some guidelines on how this documentation is generated:
22 
23  - doxygen is used to parse the documentation, thus everything works like in C code:
24  - use double comment (##, like /// in C) to start a doxygen line.
25  - comments must be placed before the declaration of the function they comment, or the class creation.
26  - all [special commands](https://www.stack.nl/~dimitri/doxygen/manual/commands.html) are available. For declaring
27  a command, \\ or \@ character can be used: for backward compatibility with previously written code, please use
28  \@ symbol only in python. Here is an example ({at} stands for \@ character):
29 
30  ## {at}brief Gives the answer.
31  ## {at}details This function gives the answer life, the universe and everything.
32  ## {at}note Not everybody might be psychologically ready for the answer.
33  ## {at}param question The question input.
34  ## {at}return 42
35  def answer(the_question):
36  return 42
37 
38  - File parsed: the following file are parsed:
39  - all python files (files with extension .py) in jiminy, **except** __init__.py and setup.py files and
40  files in a build (sub)directory or in a unit (sub)directory.
41  - all files in a scripts directory, regardless of extension (command line scripts have no extensions). Files
42  without extensions are considered to be python scripts.
43  - .h files inside a directory named python.
44 
45  - To add a file to the [Command Line Scripts](./group__scripts.html) page, simply add `\@ingroup scripts` at the top of
46  the file.
